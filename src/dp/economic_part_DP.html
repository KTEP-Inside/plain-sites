Экономическая часть ДП

Экономическая часть дипломного проекта структурно состоит из трех частей:

  • технико-экономическое обоснование проекта;
  • расчет затрат на разработку ПП;
  • оценка результата (эффекта) от использования (или продажи) ПП.

Технико-экономическое обоснование разработки ПП

В данном разделе необходимо дать краткую характеристику разработанного программного продукта, какие функции оно выполняет, какие задачи решает, где и для каких целей применяется. Необходимо провести анализ разработанного в дипломном проекте программного продукта с экономической точки зрения и обосновать актуальность (целесообразность) разработки, а также внедрения на конкретном предприятии (организации).

Обязательно следует указать, кто является потенциальным пользователем ПП и существует ли потребность в нем.

Возможны три основных варианта:

  1. Разработка (совершенствование) программного обеспечения для собственных нужд, либо совершенствование уже существующего ПП.
  2. Разработка (совершенствование) программного обеспечения по индивидуальному заказу (например, заказ от оператора мобильной связи на разработку автоматической системы учета стоимости разговоров), либо совершенствование уже существующего ПП.
  3. Разработка программного обеспечения для свободной реализации на рынке IT (например, разрабатывается какое-либо приложение для мобильных телефонов, которое свободно может приобрести любой желающий).

Для обоснования экономической целесообразности разработки и внедрения в практическую деятельность предлагаемой системы (продукта) необходимо провести сравнительный анализ имеющихся аналогов.
